after

after tag
after *

Is true when an element has a following sibling element with tag tag, false otherwise.

Use the wildcard (“*”) to match any element tag.

This predicate can be used as a point of view.

For example:

<h1>New Brunswick Bucket List</h1>
<p>Never negate the boundless beauty of these places:</p>
<ul>
  <li>Edmunston</li>
  <li>Saint John</li>
  <li>Kouchibouguac</li>
</ul>

For the h1 element, the following predicates give true:

after ul
after p

When used as a point of view, after applies to the first sibling named tag.

For example:

<ul>
  <li>Edmunston</li>
  <li>Saint John</li>
  <li>Kouchibouguac</li>
</ul>

For the li element containing “Edmunston”, the following gives true:

after-contains li "Saint"

But the following gives false:

after-contains li "Kouchi"

Even though there is an li element after “Edmunston” that contains “Kouchi”, the latter is not the first li element after “Edmunston”.
