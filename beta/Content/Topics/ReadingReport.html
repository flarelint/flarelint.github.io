Reading a Report

FlareLint saves your report in the same folder as your project. Its file name is FlareLintReport.html. FlareLint does not resolve references to variables, snippets, cross-references, conditional text, and so on. So interpret its messages with that in mind.

A FlareLint report has a header and a list of files in the project that caused errors or warnings.

The header lists the file name of the project, date that FlareLint generated the report, the Windows user name of the person who started FlareLint, and the number of errors and warnings.

Each Flare file in the list of files has at least one error or warning message. The report does not list files that result in no errors or warnings.

Tip: To view a Flare file in a new browser window or tab, click its file name.

Each error and warning describes the following:

  • type of message, error or warning
  • XML element in which it occurred
  • contextual information from the element
  • error or warning message itself

Error messages have red vertical bars. An error describes a problem that will give unintended output or cause production failure.

Warning messages have yellow vertical bars. A warning describes something suspicious that needs your further investigation. Or it describes a situation that is not a show-stopper but could result in better content if it is fixed.

Tip: Establish guidelines on handling errors and warnings. For example, your team might decide that all errors must be fixed but warnings are acceptable.
