Comment Section

Contains information, annotations, notes, reminders, and other information for you and other people to read. FlareLint ignores the content of a comment section. The comment is only visible in a rule file, it is not included in a FlareLint report. A comment section is optional. If a rule has a comment section, it must be at the beginning.

Example of comment section:

comment:

This rule checks for the presence of an h1 header. 
See other rules for checking for the position of 
the header in the topic body.

Another example of a comment section:

comment:

Makes sure that headers always start with the 
product name.
