Test Section

Verifies that the element that was matched by a when section meets the rule’s criteria. A rule file contains at least one test section and can contain many. The test sections must be after the when sections and before the message section.

After succeeding in matching a file with the extensions section, then finding an element in the file with a when section, FlareLint then checks to see if the element meets the criteria specified in the test sections.

For example, to match paragraph elements in list items and test that they end with a period (“.”):

when: self p and parent li
test: self-ends p "."

FlareLint applies each test section in the order it appears in the rule file. FlareLint stops at the first test section that is true. If the element fails all the test sections in a rule, then the element has broken the rule and FlareLint inserts the rule's message in its report.

For example, your team does not allow empty paragraphs. The self-empty predicate gives true for an element that contains no text. However, your team agrees that an empty paragraph in a table cell is acceptable, and that a paragraph can contain elements that represent content, such as images, snippets, and variables. To test that all paragraphs are appropriately and completely empty:

when: self-empty p

test: ancestor td
test: descendant img
test: descendant MadCap:snippetText
test: descendant MadCap:variable
