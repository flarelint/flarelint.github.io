Managing Rules

FlareLint comes with a default set of rules. You might find that some are too general, too specific, or not relevant for your projects. Or you might want to confirm which rules FlareLint uses.

You can view which rules are active and specify which rules to apply. You can also define a standardized set of rules for your team.

The rules that FlareLint applies are in your personal rule folder, %APPDATA%\FlareLint.

The %APPDATA% variable is defined by Microsoft Windows to refer to your personal application data. For example, in Windows 7

%APPDATA%\FlareLint

expands to

C:\Users\username\AppData\Roaming\FlareLint

where username is your user account on your computer.

The first time you run FlareLint, it first checks to see if your personal rule folder exists. If it does not, FlareLint installs its default rules. The files for default rules have names starting with flarelint_. FlareLint does not delete, modify, or overwrite the rules that are already in your personal rule folder. Like any other rules, you can add and remove default rules to and from your personal folder.

Tip: Establish a fixed set of rules for all your team to use by storing their Python files in a shared folder. Require each team member to copy these files to each of their personal rule folders.
