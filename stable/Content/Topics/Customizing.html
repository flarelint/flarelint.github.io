Customizing and Distributing

You can customize FlareLint and distribute your improvements.

FlareLint is open source so you have full access to its source code. For example, you can add, modify, or delete rules. You can integrate with other tools. You can customize HTML output or generate reports in new formats.

To customize FlareLint, you need knowledge of Python 3, HTML, and CSS. The source files for FlareLint and this guide are in the src folder, described in the following table:

File or folder in src Description

FlareLint.bat

Microsoft Windows batch file to invoke the Python interpreter with the Python flarelint module.

README.txt

Introductory text.

LICENSE.txt

The full text of the Common Development and Distribution License Version 1.0 (CDDL-1.0).

build.py

Python module to build this user guide, build the Python FlareLint package, and bundle them into the FlareLint installer, FlareLint_2-0b.zip.

setup.py

Python module that specifies the parameters for building the Python FlareLint package.

doc folder

MadCap Flare project for this user guide.

flarelint folder

Source files for the Python FlareLint package. For more information on these files, see the comments in each file.

flarelint\rules folder Source files for the default rules that FlareLint installs when your personal rules folder is empty. Your personal rules file is %APPDATA%\FlareLint.

To distribute FlareLint, please follow the terms of the Common Development and Distribution License Version 1.0 (CDDL-1.0).
